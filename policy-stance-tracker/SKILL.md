# 政策文本立场追踪 Skill

## 这个skill解决什么问题

国际关系研究里经常需要回答"某国在某议题上的立场是不是变了、什么时候变的、为什么变"。
直接把一堆文本丢给AI做摘要，最容易出两个问题：

1. **断章取义**：AI摘出一句"强硬表态"，但不说这句话是在什么场合、对谁说的、前后文是什么——脱离语境的表态经常被过度解读成"立场转变"。
2. **虚假的转折点**：AI为了让叙事显得有变化，容易夸大或编造某个时间点的"立场突变"，而实际上原文证据并不支持。

这个skill的核心不是让AI去自动生成一条好看的立场时间线，而是**在AI每一次想要标记"立场转变"的地方，强制停下来要求原文证据和人工确认**——这是本skill与普通摘要工作流的区别。

## 适用场景

- 追踪特定国家/领导人对某个议题（贸易、安全、气候、多边机制改革等）的立场变化
- 分析领导人个人风格（如交易主义、意识形态导向）如何影响一个双边/多边关系
- 为论文的"因变量"（立场/政策变化）提供有出处、可核查的证据链，而不是AI凭印象给的结论

## 三智能体协作架构

本skill不是一个"AI从头到尾自动跑完"的单一流程，而是三个**分工不同、会互相提问**的智能体角色，外加一个纯工具脚本。三个角色的划分依据是"谁对谁的输出负责审查"，这样才构成真正的协作，而不是同一个AI换个人称说三遍。

**无需API、无需编程即可运行**：三个角色都可以由同一个对话式AI（如Claude）按顺序扮演，使用者手动把上一个角色的输出复制给下一个角色作为输入即可。这个"手动传递"本身就是"人工把关"要求的一部分，不是权宜之计。详细的无代码操作步骤见 `references/three_agent_workflow.md`。

### Agent A · 摘录与语境智能体（Extraction & Context Agent）
- **输入**：原始材料（讲话稿、联合声明、政府白皮书、可靠媒体对官员表态的报道等）
- **输出**：结构化条目，含 `date` / `actor` / `content_paraphrase`（转述，禁止逐字复制原文超过15个词）/ `source`（可回溯出处）/ `context`（occasion / audience / surrounding_events / possible_misreading）
- **自主性**：自行判断哪句话构成可编码的"表态"
- **反应性**：找不到一手或高质量二手来源时主动停止，标记"证据不足"，不得靠AI记忆填补
- **停止条件**：完成一批材料的摘录，或遇到证据不足时逐条停下

### Agent B · 编码与审计智能体（Stance-Coding & Audit Agent）
- **输入**：Agent A 产出的结构化条目
- **输出**：打分后的完整条目 + 疑似立场转变标记 + 休谟三原则检查结果
- **量表**：本skill默认"交易主义强度"量表（0-3分，见 `references/methodology.md`，可替换为其他议题的量表）

| 分值 | 含义 |
|---|---|
| 0 | 强调共同价值观/制度性承诺，不提具体对价 |
| 1 | 提及互惠，但仍以联盟/伙伴关系语言为主 |
| 2 | 明确要求具体让步（关税、市场准入等）作为交换条件 |
| 3 | 提出具体金额/份额的单边要价，或将安全承诺与经济让步直接挂钩 |

- **社会性**：如果 A 给的语境信息不足以支撑打分，B 有权把该条目**打回给 A** 要求补充，而不是自己脑补——这是A、B之间真正的双向互动，不是单向流水线
- **主动性**：当相邻两个已验证节点分值差 ≥ 1 时，B 主动标记 `⚠ 疑似立场转变`，列出两侧原文证据，并说明"这个变化会不会只是场合/听众不同造成的语域切换，而不是立场真变了"（对应休谟三原则的"排除混淆"）
- **停止条件**：疑似转变点必须等待人工回复"确认"或"存疑"后才能写入最终结论，B 不能自己拍板

### Agent C · 信源可信度与词频信号智能体（Source Credibility & Signal Agent）
这是本次新增的角色，也是词云统计工具（`assets/tools/trump_wordcloud_tool.html`）发挥作用的地方。
- **输入**：原始语料 + 词云工具跑出的词频统计数据（交易主义词汇/盟友词汇的出现频次）
- **输出**：
  1. 对每条表态的信源类型给出可信度权重建议（官方文件 > 联合声明 > 媒体转述 > 回忆录/爆料转述），供 B 打分时参考权重
  2. 根据词频异常（比如"deal""pay""fair"等词在某段时期密集出现），**主动**把线索反馈给 A，提示"这段时期的文本值得优先精读"
- **主动性最强**：不需要被指令触发，扫描到高频异常信号就主动生成线索
- **社会性**：C 的线索会反馈给 A（形成"C→A"的回路），C 对信源的权重判断也会被 B 参考（形成"C→B"的回路），A、B、C 三者之间是网状协作，不是单线流水线
- **停止条件**：只给建议和权重参考，不直接修改任何条目的 `verified` 或 `stance_score`，最终决定权始终在人工

### 工具（不算智能体，是纯确定性脚本）
- `scripts/build_timeline.py`：把 Agent A/B 产出的、已经过人工确认的条目画成时间线图和报告，只是执行工具，不涉及判断，因此不计入"智能体"数量
- `assets/tools/trump_wordcloud_tool.html`：纯前端词频统计（正则匹配+固定词表），本身没有AI判断，是 Agent C 的信息来源，不是 Agent C 本身

## 停止条件总览
- Agent A：证据不足时逐条停
- Agent B：疑似转变点必须停，等人工确认
- Agent C：给完线索/权重建议即停，不越权改结论
- 全流程：时间范围内所有表态都完成 A→B 的确认闭环，或人工主动喊停

## 防幻觉硬性规则

- 每条表态必须有可回溯来源，AI凭训练记忆"回忆"出的表态一律不采信，需重新检索确认
- 转述禁止逐字照抄超过15个词
- 一个来源只能被引用一次原句（详见 references/methodology.md 中的引用规范）
- 如果同一事件有多个信源说法冲突，两个都列出，不擅自取舍

## 终止条件

- 时间范围内所有表态都完成 Agent A→B 的确认闭环（Agent C 的建议已被参考或明确搁置），或
- 用户主动喊停 / 表示当前证据已经够用于论文这一部分
