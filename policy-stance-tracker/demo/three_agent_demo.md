# 三智能体工作流演示：A → C → B 完整链路

本演示用 `demo/trump_japan_entries.json` 里"驻日美军费用要价（据博尔顿回忆录）"这一条，
展示三个角色之间是怎么互相提问、最终仍然保持"待核实"状态的——这正是本skill要证明的核心能力：
**AI不会因为某条信息劲爆、好用，就自动把它写进结论。**

---

## Step 1：Agent A 的输出（摘录与语境）

```json
{
  "date": "2019-2020",
  "actor": "特朗普政府（据博尔顿回忆录披露的驻日美军费用谈判要价）",
  "content_paraphrase": "据前国家安全顾问博尔顿回忆录披露，特朗普政府在《特别措施协定》续约谈判中要求日本将驻日美军费用分摊提高到每年约80亿美元，较此前约20多亿美元大幅增加",
  "source": {"type": "memoir", "citation": "SCMP + CRS报告RL33436交叉印证"},
  "context": {
    "occasion": "非公开谈判中的内部要价，经前官员回忆录披露",
    "audience": "谈判对手方（日方团队），并非面向公众的正式宣示",
    "possible_misreading": "回忆录披露的谈判开价可能是策略性极端开价，不完全等同于对外公开宣示的正式政策立场"
  }
}
```

Agent A 在这里已经主动做了一件事：把"信源类型是回忆录"这个风险点写进了 `possible_misreading`，
没有把它包装成一条板上钉钉的"官方立场"。

---

## Step 2：Agent C 的输出（信源可信度 + 词频信号）

用词云工具跑一段描述这一时期交易主义话语模式的演示文本（转述而非逐字引用原始发言），得到：

| 指标 | 数值 |
|---|---|
| 总词数 | 64 |
| 交易主义词命中 | pay, cost, troops, burden, unfair, trade, deal, reciprocal, tariffs, deficit, rip, protect（各1次，合计12次）|
| 盟友/地域词命中 | japan(2), troops(1), bases(1), alliance(1) |
| 交易词占比 | 18.8% |

**Agent C 的分析意见**：

> 信源可信度权重建议：这条属于"回忆录转述"类信源，建议权重设为四档中最低一档（低于官方文件、联合声明、主流媒体直接报道）。
> 词频信号：本时期文本中 burden / pay / cost / rip 等词密集共现，说明"驻军费用商品化"这个话语框架在同时期的其他（可公开验证的）表态里也有迹可循，值得 Agent A 回头去找 2019-2020 年是否存在**公开场合**的类似表态，用来交叉印证博尔顿回忆录里这条要价是否只是孤例。

这里 Agent C 做了两件事：一是给了权重参考（供B用），二是主动给A提了新的检索方向（"去找公开场合的类似表态"）——这就是 C→A 的反馈回路。

---

## Step 3：Agent B 的输出（编码与审计）

> 综合 Agent A 的语境标注和 Agent C 的权重建议，本条目暂不建议直接编码为 `verified: true`：
>
> 1. **信源单一**：目前只有回忆录一个信源类型，Agent C 建议的交叉印证（找公开场合的同期表态）尚未完成
> 2. **场合特殊**：这是非公开谈判中的策略性开价，休谟三原则里"排除混淆"这一项无法通过——不能排除这只是谈判筹码而非最终政策立场
> 3. **建议打回 Agent A**：请补充检索 2019-2020 年是否有公开声明/国会证词提到类似的费用分摊要求，作为交叉印证
>
> **本条目暂时保持 `verified: false`，进入"待人工核实"清单，不计入正式立场时间线。**

---

## 这个演示说明了什么

如果没有 Agent C 的介入，Agent B 单独审查这条"每年80亿美元"的爆料，很可能因为它戏剧性强、数字具体，
被误判为一个"确凿的立场高点"直接采信。但 Agent C 从信源类型和词频交叉验证两个角度都提出了保留意见，
Agent B 因此选择了更保守的处理方式——**这正是三个角色互相牵制、而不是一个AI自我复述三遍**的体现，
也是这条记录在最终 `build_timeline.py` 生成的报告里始终待在"待人工核实清单"而不是正式时间线上的原因
（可对照 `demo/trump_japan_timeline_report.md` 第三部分）。
