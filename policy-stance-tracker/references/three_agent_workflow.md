# 三智能体工作流：无代码操作指南

写给不会编程、不打算调用API的使用者。整个流程只需要一个能对话的AI（如Claude网页版/App）+ 一个能打开html的浏览器，不需要信用卡，不需要写一行代码。

## 核心思路

A、B、C 三个"智能体"其实是**同一个AI在不同时刻扮演的三个角色**，靠你手动把上一步的输出复制给下一步当输入，从而把它们"串"起来。
这不是偷懒的替代方案——真正决定这是不是"多智能体协作"的，是**角色之间有没有明确分工、会不会互相质询、能不能在关键点停下来**，而不是有没有调用付费API。你手动搬运数据的动作，恰好扮演了"工作流编排"和"人在每一步把关"这两个角色。

## 准备工作

1. 打开一个和Claude的对话窗口（就是你现在用的这个）
2. 收集好你要分析的原始材料（讲话稿、联合声明、新闻报道等）
3. 用浏览器打开 `assets/tools/trump_wordcloud_tool.html`（双击文件即可，不需要联网）

## 第一步：运行 Agent A（摘录与语境）

在对话里这样开头：

> 请你现在扮演"Agent A · 摘录与语境智能体"。任务规则见 SKILL.md 的 Agent A 部分。
> 以下是原始材料：[粘贴你的文本]
> 请按 assets/entry_template.json 的结构，输出结构化条目（不要打分，Step留给Agent B）。

AI会输出若干条 JSON 格式的条目，**你把这段输出保存下来**（复制到一个文本文件或直接留在对话里）。

## 第二步：运行 Agent C（先于B运行，给B提供参考）

在浏览器里打开词云工具，把同一批原始材料粘进输入框，点分析，会得到词频统计和高频词列表。
截图或者把统计数字抄下来，然后在对话里另起一段：

> 请你现在扮演"Agent C · 信源可信度与词频信号智能体"。任务规则见 SKILL.md 的 Agent C 部分。
> 这是词云工具跑出的词频数据：[粘贴/描述统计结果，比如"deal出现12次，pay出现5次，集中在9月"]
> 这是Agent A摘录的条目：[粘贴第一步的输出]
> 请给出：①每条条目建议的信源可信度权重；②有没有词频异常提示某个时间段的文本值得重新精读。

AI会给你一份"权重建议+线索提示"。**这份输出你也保存下来**，作为第三步的参考材料之一。

## 第三步：运行 Agent B（编码与审计）

另起一段对话：

> 请你现在扮演"Agent B · 编码与审计智能体"。任务规则见 SKILL.md 的 Agent B 部分。
> 这是Agent A的条目：[粘贴]
> 这是Agent C给出的信源权重建议：[粘贴第二步的输出]
> 请对每条打分（交易主义强度量表见 references/methodology.md），并且：如果某条语境信息不够支撑打分，请明确指出需要把它"打回给Agent A"补充哪些信息；如果发现相邻节点分值跳变≥1，请单独列出来提示我需要人工确认。

AI会输出打分结果 + 需要打回A的条目 + 需要人工确认的疑似转变点。

## 第四步：你自己做人工确认

对着Agent B列出的"疑似转变点"和"信源存疑条目"，你自己判断：
- 原文证据是否真的支持这是一次立场转变？
- 回忆录/媒体转述类的信源，要不要采信？

确认后，把最终版本的条目整理进一个JSON文件（参照 `assets/entry_template.json`），把确认过的条目 `verified` 改成 `true`。

## 第五步：生成时间线图

这一步如果你不会用命令行，**可以直接把整理好的JSON条目发给我（Claude），让我在对话里帮你运行 `scripts/build_timeline.py` 生成图表**，这不需要你自己动手写代码。

## 一次完整流程大概长这样

```
你：[粘贴原始文本] 请以Agent A身份处理
AI：[输出结构化条目]

你：[打开词云工具跑出统计] 请以Agent C身份分析
AI：[输出权重建议+线索]

你：[把A和C的输出都粘贴进来] 请以Agent B身份打分和审计
AI：[输出打分结果+疑似转变点，可能要求打回A补充]

你：（人工核实疑似转变点，确认或存疑）

你：这是我确认后的最终条目 [JSON]，帮我生成时间线图
AI：[运行脚本，产出图表和报告]
```

全程不需要注册开发者账号、不需要信用卡、不需要写代码——你做的事情就是"复制、粘贴、判断"，但这恰恰是整个作业里"控制力"这条主线最看重的部分：AI负责执行，你负责判断和把关。
